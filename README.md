# knip-dynamic-import-demo\n\nDemonstrates how **Knip v6** treats `return import('./Module')` differently from **Knip v5**, causing false positives (and potential false removals) in React + React Router v5 monorepos that use function-based code splitting.\n\n## The problem\n\nReact Router v5 code-splitting often looks like this:\n\n```ts\n// BrokenLoader.ts\nexport function loadApp() {\n  return import('./AppBootstrap');\n}\n```\n\nIn **Knip v5** (TypeScript compiler backend), this worked fine тАФ the TS compiler followed all reachable files automatically, so `AppBootstrap` and everything it imports were included in the analysis.\n\nIn **Knip v6** (oxc-parser/oxc-resolver backend), this `return import()` pattern is handled by the catch-all `handleImportExpression` visitor, which registers the import with `IMPORT_FLAGS.OPAQUE`. That means:\n\n> Knip sees the import exists, but does **not** traverse into `AppBootstrap` or any of its transitive imports.\n\n### What breaks\n\nAfter running `npm run knip` you will see false positives like:\n\n- `AppBootstrap.tsx` тАФ reported as an unused file or its exports flagged as unused\n- `src/routes/HomeRoute.tsx` тАФ `HomeRoute` export flagged as unused\n- `src/routes/AboutRoute.tsx` тАФ `AboutRoute` export flagged as unused\n\nThese are **false positives**. The files are reachable at runtime, but Knip v6 can't see that through the `return import()` pattern.\n\n## The working pattern\n\n```ts\n// WorkingLoader.ts\nexport const loadAppWorking = () => import('./AppBootstrapWorking');\n```\n\nAn arrow function with a direct `import()` expression is handled by Knip v6's `handleVariableDeclarator`, which **does** traverse into `AppBootstrapWorking`. No false positives.\n\n## Repo structure\n\n```.\nsrc/\n  index.tsx                  # Entry point\n  BrokenLoader.ts            # тЭМ return import() тАФ Knip v6 treats as opaque\n  WorkingLoader.ts           # тЬЕ () => import() тАФ Knip v6 traverses correctly\n  AppBootstrap.tsx           # Loaded by BrokenLoader тАФ falsely flagged\n  AppBootstrapWorking.tsx    # Loaded by WorkingLoader тАФ correctly tracked\n  routes/\n    HomeRoute.tsx            # Falsely flagged as unused via BrokenLoader\n    AboutRoute.tsx           # Falsely flagged as unused via BrokenLoader\n```\n\n## Reproduce\n\n```bash\nnpm install\nnpm run knip\n```\n\n## Workarounds\n\n**Option 1 тАФ Change to arrow function syntax:**\n```ts\n// Before\nexport function loadApp() {\n  return import('./AppBootstrap');\n}\n\n// After\nexport const loadApp = () => import('./AppBootstrap');\n```\n\n**Option 2 тАФ Add the file as an explicit entry in `knip.json`:**\n```json\n{\n  "entry": ["src/index.tsx", "src/AppBootstrap.tsx"]\n}\n```\n\n**Option 3 тАФ Ignore with a JSDoc tag** on the exports in `AppBootstrap.tsx`:\n```ts\n/** @public */\nexport default function AppBootstrap() { ... }\n```\n\n## Related\n\n- [Knip v6 release notes](https://knip.dev/blog/knip-v6)\n- [Knip source: `handleImportExpression`](https://github.com/webpro-nl/knip/blob/main/packages/knip/src/typescript/visitors/imports.ts)\n- [Knip fixtures: imports-opaque](https://github.com/webpro-nl/knip/tree/main/packages/knip/fixtures/imports-opaque)
